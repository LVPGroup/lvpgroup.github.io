theory PP_DM
  imports Main
begin


(*
Data model is the formal specification of security assets.
In ProveriT, ten assets are formalized as parameterized types
in Isabelle/HOL.

The data model is specified in a parameterized
way, which means that the assets are defined as abstract
data types. For example, we model TEE persistent data as
â€²tee_data without explicitly describing the concrete data
structure.
*)

record 
('tee_id,'instime,'rng,'ta_data,'ta_code,
    'tee_runtime,'tee_data,'initcode,'firmware,'SoC) Data_Model = 
        TEE_identification::'tee_id
        (*TEE identification data that is globally unique among all TEEs whatever the manufacturer, vendor or integrator.
        This data is typically stored in the OTP memory of the SoC. The TEE identifier is intended to be public 
        and exposed to any software running on the device, not only to Trusted Applications. 
        GlobalPlatform TEE should return this identification through a standard TEE Internal Core API.*)
        TA_instance_time::'instime
        (*Monotonic time available during TA instance lifetime. 
        Not affected by transitions through low power states. Not persistent over TEE reset or TA shut-down.*)
        RNG::'rng
        (*Random Number Generator.*)
        TA_data_and_keys::'ta_data
        (*Data and keys managed and stored by TAs using the TEE security services. 
        Data and keys are owned either by the user (the owner of the TEE-enabled device) or by the TA service provider. 
        This data is typically stored in external non-volatile memory which is shared with the REE and potentially accessible by it.*)
        TA_code::'ta_code
        (*The code of the installed Trusted Applications. 
        This data is typically stored in external non-volatile memory which is shared with the REE and potentially accessible by it.*)
        TEE_runtime_data::'tee_runtime
        (*TEE runtime data includes execution variables, runtime context, etc. This data is stored in volatile memory.*)
        TEE_persistent_data::'tee_data
        (*TEE persistent data includes cryptographic keys (for instance keys to authenticate TA code) and TA properties. 
        This data is typically stored in external non-volatile memory which is shared with REE and potentially accessible by it.*)
        Initialisation_code_and_data::'initcode
        (*Initialisation code and data (for instance cryptographic certificates) 
        used from device power-on up to the complete activation of the TEE security services.*)
        TEE_firmware::'firmware
        (*The TEE binary, containing TEE code and constant data such as versioning information. 
        This asset is typically stored in external non-volatile memory which is shared with REE and potentially accessible by it.*)
        TEE_storage_root_of_trust::'SoC
        (*The root of trust of the TEE storage that is used to bind the stored data and keys to the TEE. 
        This root of trust is typically derived from hardware secrets stored in the OTP memory of the SoC. 
        Confidentiality is generally ensured by the fact that the asset remains inside the SoC.*)


end
